# Project Context — Study Load Optimizer (Syllabi)

A full-stack academic workload management app for University of Waterloo students. Users upload syllabi, track assessments, log study sessions, and generate optimized study plans.

## Architecture

- **Frontend**: Next.js 16 (App Router), React 19, TypeScript, Tailwind CSS 4, shadcn/ui
- **Backend**: Python 3.11+, FastAPI, SQLAlchemy 2.0, Alembic
- **Database**: PostgreSQL 15
- **Auth**: JWT (python-jose + bcrypt)
- **AI**: Google Gemini API for syllabus PDF parsing
- **Deploy**: Vercel (frontend), Docker (backend + DB)

## Data Model

```
User
 └── Term (name, start/end dates, sort_order, is_archived)
      └── Course (code, name, color)
           ├── Assessment (title, type, due_date, weight, is_completed)
           └── StudySession (date, duration_minutes, notes, is_completed)

User
 └── Availability (day_of_week, start_time, end_time)

StudyBlock (generated by planner — links assessment + time slot)
```

## Backend API (`/api/v1`)

| Module           | Prefix             | Key Endpoints                                                 |
| ---------------- | ------------------ | ------------------------------------------------------------- |
| Auth             | `/auth`            | register, login, me                                           |
| Terms            | `/terms`           | CRUD, reorder                                                 |
| Courses          | `/courses`         | CRUD, upload-syllabus (Gemini), delete assessments            |
| Assessments      | `/assessments`     | CRUD, bulk create, upcoming, toggle-complete                  |
| Study Sessions   | `/study-sessions`  | CRUD, toggle-complete                                         |
| Availability     | `/availability`    | CRUD                                                          |
| Plan             | `/plan`            | generate (creates optimized study blocks)                     |
| Analytics        | `/analytics`       | study-time breakdown, grade-forecast                          |

## Key Services

- **Planner** (`services/planner.py`) — Generates study blocks by mapping upcoming assessments to available time slots, prioritizing by due date and weight.
- **Waterloo Parser** (`utils/waterloo_parser.py`) — Parses UW course syllabus PDFs/HTML to extract assessment names, dates, and weights using regex + BeautifulSoup + Gemini.

## Environment Variables

| Variable         | Description                                    |
| ---------------- | ---------------------------------------------- |
| `DATABASE_URL`   | PostgreSQL connection string                   |
| `SECRET_KEY`     | JWT signing secret                             |
| `DEBUG`          | Enable debug mode                              |
| `GOOGLE_API_KEY` | Gemini API key (syllabus parsing)              |
| `GOOGLE_MODEL`   | Gemini model (default: gemini-2.0-flash)       |

## Deployment

- **Frontend**: Deployed on Vercel at `study-load-optimizer.vercel.app`
- **Backend**: Dockerized with `docker-compose.yml` (PostgreSQL + FastAPI)
- **CORS**: Configured for localhost:3000 and the Vercel production/preview URLs
